## 🛑 为什么重构这个项目？

曾经的 BUCTBASE 是一潭死水。我们受够了大学里那种“优质资料靠私下乞讨，垃圾重点在群里乱飞”的病态现状。

前人踩过的坑，后人没有理由再踩一次。本项目致力于打破院系壁垒与信息垄断，收集并沉淀北化（BUCT）最硬核、最干货的课程资料。**这里不是垃圾场，我们只收录能真正帮你拿高分、懂原理、过关斩将的核心资产。**

## 📂 核心资产 (Core Assets)

目前已收录以下高质量资料（按学院与课程严格分类）：

* **⚡ 核心专业课：** 计科 / 化工 / 过装 / 公管 等核心课程的高分笔记与复核资料。
* **🎯 历年真题库：** 拒绝模糊不清的残卷，只提供高清晰度、带靠谱解析的试卷。
* **🛠️ 避坑指南针：** 前人血泪写就的选课指南、教师风格评测、大作业高分套路。
* **💻 硬核代码库：** 历年优秀的大作业/课程设计源码及实验报告（仅供参考思路，严禁无脑抄袭）。

## ⚔️ 贡献法则 (Hardcore Contribution Guide)

**我们极其渴望高质量的贡献，但我们对垃圾内容零容忍。**

如果你只是想白嫖，默默下载走人即可，别来捣乱。如果你想成为这个硬核圈子的一员，请严格遵守以下法则：



### 1. 我们拒绝什么？ (What we REJECT)
* ❌ **劣质格式：** 拒绝 .doc / .rtf 等封闭格式，所有文档优先提交 Markdown (`.md`) 或 PDF。
* ❌ **无脑堆砌：** 拒绝未经整理的几百张连拍照片，请使用扫描软件合成清晰的 PDF。
* ❌ **侵权内容：** 严禁上传任何商业付费电子书、盗版软件及答案解析。
* ❌ **无效 PR：** 提交说明（Commit Message）写得不明不白、文件命名乱七八糟的 Pull Request，将被直接关闭。

### 2. 标准提交流程 (How to PR)
1.  **Fork** 本仓库到你的个人主页。
2.  严格按照 `[学院]/[课程名]/[资料类型]` 的目录结构放置文件。**文件命名必须清晰**（例：`2023_计算机网络_期末A卷_附答案.pdf`）。
3.  发起 **Pull Request**。
4.  等待 Review。如果资料质量不达标，你的 PR 会被打回。

## 🚀 网络与下载加速

别再抱怨 GitHub 慢了，技术人要学会自己解决网络问题。如果直连下载困难，请使用以下合法加速手段：

* **手段一：** 将 `github.com` 替换为加速镜像站点，如 `kgithub.com` 或 `hub.nuaa.cf` 进行访问和下载。
* **手段二：** 使用 [Watt Toolkit (原Steam++ )](https://steampp.net/) 进行本地网络环境加速。
* **手段三：** 复制目标文件的链接，使用 [GitHub Proxy](https://ghproxy.com/) 代理下载。
